# Tailwind Migration + Mobile-Friendly Header Redesign Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Migrate `runlog-website` from hand-rolled `register/style.css` to Tailwind v4 across all 15 HTML pages, and replace the burger nav with a wrap-on-narrow header where every link is visible at every viewport width.

**Architecture:** Tailwind v4 CLI builds `src/input.css` → `dist/style.css` (committed). The shared theme is declared with `@theme` plus a `prefers-color-scheme: dark` media query that overrides the same CSS custom properties. Recurring patterns (`.cta`, `.card`, `.note`, etc.) stay as named classes defined via `@apply` in `input.css`; per-page tweaks become inline utilities on markup or additional `@apply` rules. Cloudflare deploy contract unchanged — still static-asset deploy via wrangler, no CI build step.

**Tech Stack:** Static HTML, Tailwind CSS v4 (via `@tailwindcss/cli`), Cloudflare Pages (unchanged), Wrangler.

**Spec:** `docs/superpowers/specs/2026-05-03-tailwind-header-redesign-design.md`

## Class Rename Reference

The migration renames a few classes to avoid Tailwind utility collisions
or to flatten descendant-selector patterns. Apply these renames
consistently across every page:

| Old markup                                    | New markup            | Reason                                               |
|----------------------------------------------|----------------------|------------------------------------------------------|
| `<div class="grid tease ...">`                | `<div class="grid-tease ...">` | Avoid colliding with Tailwind's `.grid` utility |
| `<div class="grid">`                          | `<div class="grid-cards">`     | Same reason                                          |
| `<div class="grid pricing">` (pricing page)   | `<div class="grid-cards">`     | Drop `pricing`; descendant rule replaced by global `.price` |
| `<div class="grid agents">` (agents page)     | `<div class="grid-cards">`     | Drop `agents`; cards get `class="card agents-card"` |
| `<footer>` (bare)                             | `<footer class="site-footer">` | Scoped `.site-footer` replaces bare `footer` selector |

Exact occurrences in the current repo (use `grep` to locate during the
migration tasks):
- `index.html:256` and `index.html:281` — both `grid tease` → `grid-tease`
- `agents/index.html:169` — `grid agents` → `grid-cards`
- `pricing/index.html:164` — `grid pricing` → `grid-cards`
- `trust/index.html:120` — `grid` → `grid-cards`
- (No grids in `quickstart/`, `why-verification/`, blog pages.)
